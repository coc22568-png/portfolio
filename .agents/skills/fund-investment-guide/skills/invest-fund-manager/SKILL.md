# 基金经理研究系统

> 业绩先说话，季报验逻辑，年检不行就换。

---

## 工作模式

收到请求后判断模式，直接执行：

| 模式 | 触发场景 | 执行 |
|------|---------|------|
| **A：筛选新标的** | 没有具体目标，想找某策略的经理 | 生成 AI 筛选提示词 |
| **B：深度分析** | 已锁定某经理/基金，做买入前或年检尽调 | 六维分析框架 |
| **C：年检** | 季报/年报出来了，对持仓做健康检查 | 对照档案逐一核查 |
| **D：档案管理** | 用户想新建或更新经理档案 | 读取 references/manager-profile.md |

---

## 模式A：筛选新标的

用户没有具体目标，想找某类策略的基金经理。

**策略路由**：

| 关键词 | 参考文件 |
|-------|---------|
| 红利、股息、现金流 | `references/prompt-红利价值.md` |
| 价值、低估、均衡、深度价值 | `references/prompt-主动价值.md` |
| 量化、超额、CTA | `references/prompt-量化.md` |
| 指增、指数增强 | `references/prompt-指增.md` |
| 港股、恒生、央企港股 | `references/prompt-港股红利.md` |

读取对应文件，生成可直接用于 AI 搜索的筛选提示词，交给用户去查询。

---

## 模式B：深度分析（六维框架）

已锁定某经理/基金，做买入前评估或年检尽调。

读取 `references/manager-profile.md` 了解档案格式。

**六维分析**，按顺序逐项输出：

### 1. 近期业绩表现
- 今年以来 vs 同类基准
- 近一年 vs 同类基准
- 同类排名（前30%为及格线，连续2期后50%触发警报）

### 2. 业绩来源拆解
- 上涨靠什么？（选股/行业/风格因子/时机）
- 是否可解释、可持续，还是运气成分？

### 3. 投资框架验证
- 季报/年报里经理怎么说？
- 说的和做的是否一致？（说价值，持仓是否真的低估值？）

### 4. 风险信号扫描
- 风格漂移（价值经理开始买成长/AI）
- 换手率异常（突然翻倍）
- 基金公司层面风险

### 5. 公司层面评估
- 公司是否稳定审慎
- 有无激进营销、频繁换人记录
- 对该产品的支持力度

### 6. 决策建议
明确输出三选一：
- **继续持有**：逻辑未变，业绩达标
- **进入观察期**：有疑点，下期季报再看
- **换掉**：触发卖出条件，同时给出替换方向

---

## 模式C：年检

季报/年报发布后，对持仓经理做定期健康检查。

**卖出触发信号**：

| 信号 | 说明 |
|------|------|
| 近一年同类排名后50% | 业绩红线 |
| 季报逻辑自相矛盾 | 说不清楚为什么跌/涨 |
| 持仓风格明显漂移 | 价值经理开始买AI/成长 |
| 换手率突然翻倍 | 操作风格改变 |
| 基金经理离职 | 直接触发 |
| 基金公司重大人事变动 | 进入观察期 |

> 「不行就换」是原则，但换之前要知道换成什么。

---

## 模式D：档案管理

研究完成后，为经理建立或更新档案。详细格式见 `references/manager-profile.md`。

档案记录：投资框架、能力圈、买入理由、卖出触发条件、年检记录。

---

## 统一输出格式

```markdown
# [经理姓名] · [基金名称]（[代码]）分析

> **结论**：继续持有 / 进入观察期 / 换掉

## 核心发现（3-5点）
1. [业绩维度]
2. [逻辑维度]
3. [风险维度]

## 六维评估

| 维度 | 发现 | 评价 |
|------|------|------|
| 近期业绩 | ... | 达标/观察/警报 |
| 业绩来源 | ... | ... |
| 框架验证 | ... | ... |
| 风险信号 | ... | ... |
| 公司评估 | ... | ... |

## 决策建议
[明确的继续/观察/换掉，附理由]

## 后续观察点
- [观察点1]
- [观察点2]

---
*免责声明：本分析仅供参考，不构成投资建议。*
```

---

## 核心原则

- **业绩优先**：近期排名是入场券，逻辑验证是留场券
- **不恋战**：触发卖出条件就换，不等待、不侥幸
- **不替决策**：提供分析框架，最终决定交还用户
- **数据不足时说明局限**：没有季报就不评框架，不用猜测填充
